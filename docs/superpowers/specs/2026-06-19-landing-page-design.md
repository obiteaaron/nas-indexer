# NAS Indexer 官网设计文档

## 概述

为 NAS Indexer 项目创建一个产品官网（单页落地页），采用 Dark Gaming 设计风格，支持 GitHub Pages 独立部署和项目内嵌两种模式。

---

## 目标

- **品牌展示**：突出产品定位（NAS 文件索引 + 游戏管理一体化）
- **功能介绍**：清晰展示核心功能模块
- **快速上手**：提供简洁的安装/部署指南
- **轻量部署**：单 HTML 文件（CSS 内联），GitHub Pages 直接托管

---

## 设计规范

### 风格定位

**Dark Gaming** - 纯黑背景 + 橙色强调 + 游戏风格

| 元素 | 规格 |
|------|------|
| 主背景色 | `#18181b`（深灰黑） |
| 强调色 | `#f97316`（橙色） |
| 次背景色 | `#09090b`（纯黑）、`#27272a`（边框/卡片背景） |
| 文字主色 | `#ffffff`（白色） |
| 文字次色 | `#a1a1aa`（浅灰）、`#71717a`（中灰）、`#52525b`（暗灰） |
| 字体 | 系统字体栈（-apple-system, PingFang SC, Microsoft YaHei 等） |
| 代码字体 | Consolas, Monaco |

### 语言策略

- 中文为主，关键术语保留英文
- 保留英文术语示例：NAS、Steam API、SQLite、Docker、GitHub、MIT、Profiles

## 技术实现

### 技术选型

| 方案 | 说明 |
|------|------|
| **单 HTML 文件** | CSS 内联，无需构建，GitHub Pages 直接托管 |

### 文件结构

```
docs/
  landing/
    index.html          # 单文件官网（CSS 内联）
```

### 响应式适配

- 桌面端（>768px）：完整布局
- 移动端（≤768px）：
  - 导航链接隐藏
  - Hero 字号缩小
  - 功能卡片单列
  - 快速开始步骤单列

---

## 部署方案

### GitHub Pages

1. 将 `docs/landing/index.html` 推送到仓库
2. GitHub Pages 设置：Source = `docs` 分支 / `landing` 目录
3. 访问地址：`https://<username>.github.io/nas-indexer/`
